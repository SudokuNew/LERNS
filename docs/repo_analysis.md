# LERNS リポジトリ解析メモ

このドキュメントは、今後の機能追加要望に素早く対応するための現状把握メモです。

## 全体構成
- `index.html`: 単一ページ構成。ホーム/記録/計画表/動画/TODO の各パネルを内包。
- `css/`: レイアウト・タブ・タイマー・四字熟語カードのスタイルを分割管理。
- `js/`: 機能別スクリプト（タイマー、タブ切替、時計、TODO、四字熟語、共通関数）。
- `manifest.json` と `service-worker.js`: PWA 化（ホーム画面追加・キャッシュ）。
- `docs/`: 学習計画 PDF。
- `media/` と `assets/`: 動画・背景画像などの表示素材。

## UI/機能マップ
- ホーム (`panel-home`)
  - JST時計ウィジェット
  - 四字熟語カード（前後ナビ + インジケータ）
  - 合計学習時間表示
  - 3種類の時間管理（ポモドーロ / カスタムタイマー / ストップウォッチ）
- 記録 (`panel-log`)
  - 教科 + 分数入力
  - localStorage に学習ログ保存
- 計画表 (`panel-schedule`)
  - PDF の表示/非表示切替
- 動画 (`panel-video`)
  - セレクトボックスで YouTube 埋め込み先切替
- TODO (`panel-todo`)
  - 別スクリプトで管理

## データ保存の現状
- 学習ログキー: `lerns_logs`（`js/functions.js`）
- その他（TODO・タイマー設定など）は各機能スクリプトで個別保持

## 今後の実装で押さえるポイント
1. **DOM依存が強い**: ID 変更時は `index.html` と各 `js/*.js` の参照整合が必須。
2. **単一ページ設計**: 新機能は「新パネル追加」か「既存パネル拡張」のどちらかで統一すると保守しやすい。
3. **ローカル保存の互換性**: localStorage キー変更時は移行処理を用意する。
4. **PWA挙動確認**: リソース追加時は `service-worker.js` のキャッシュ戦略確認が必要。

## 要望対応の進め方（提案）
- 1要望ごとに以下テンプレートで実施:
  1. 変更対象（HTML/CSS/JS）を宣言
  2. 差分実装
  3. ローカル動作確認
  4. 影響範囲（他タブ/保存データ/PWA）をチェック
